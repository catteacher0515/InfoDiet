## 5. 当前风险与注意事项

### 5.1 本轮为了联调补充了本地样例数据

- 为了稳定验证日报历史、分页和失败筛选，已在本地数据库补充历史日报与推送记录样例数据
- 这些数据适合验收，不代表生产初始化策略已经收口

### 5.2 RabbitMQ 当前依赖 Docker 容器

- 当前本地 RabbitMQ 依赖 Docker Desktop + `infodiet-rabbitmq` 容器
- 如果容器停掉，异步消费链路会重新退化为连接失败

### 5.3 历史坏样本已补清理与隔离方案

- 代码层已新增坏样本隔离：
  - 内容已不存在的待推送记录不会再进入正常 RabbitMQ 入队链路
  - 入队前会先把这类记录直接终态化为失败
- 数据层已新增清理脚本：
  - `sql/cleanup_invalid_user_content_push.sql`
  - 用于逻辑删除历史坏样本及对应告警
- 这意味着后续回归不会再被同类坏样本持续污染

- 代码能力比方案更丰富
- 但产品层表达还不够收口，容易让后续维护时分不清“规则订阅”和“订阅源订阅”的边界

建议后续收口方向：

- 明确哪些属于关键词规则
- 哪些属于作者 / 频道 / 仓库订阅
- 页面文案、接口命名、数据展示统一成同一套心智模型
